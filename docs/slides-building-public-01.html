O que decidimos nesta sessão

Quatro decisões que cortaram peso.

  1. 01
    Cortamos embeddings vetoriais.
    O próprio LLM raciocina similaridade melhor que cálculo de distância.
  2. 02
    Cortamos processamento multimodal interno.
    O app host textualiza áudio/imagem/web. A gente só organiza o texto.
  3. 03
    Embedding local também saiu.
    Foco em VPS pequenas. API externa via litellm, custo irrelevante.
  4. 04
    Major Tag → Tag é eixo único.
    Sem rede de RAG por baixo. Sinopse no topo do .md guia a navegação.

Para o agente, "sócio" e "cofundador"
são parentes por padrão.
Não precisa de embedding pra isso.

Observação que destravou o design Insight da sessão

Estado atual · v0.1

O que já está rodando.

  1. ✓
    Pacote Python estruturado.
    pyproject.toml, src layout, comando sincron-brain registrado como entry point.
  2. ✓
    Vault funcional.
    .md no disco + SQLite com FTS5. CLI init cria, stats diagnostica.
  3. ✓
    MCP server com 8 tools.
    remember, list_major_tags, list_tags, read_memory, use_memories, search, sleep_now, stats.
  4. ✓
    10 providers configuráveis.
    Anthropic, OpenAI, Gemini, Voyage, Cohere, Mistral, Azure, Bedrock, Ollama, custom.
A interface que o agente recebe

Oito tools. Curtas.

remember() enfileira texto pro próximo sono
list_major_tags() árvore de temas com score
list_tags() tópicos + sinopses sob um tema
read_memory() inspecao neutra/debug
use_memories() abre conteudo + reativa no sono
search() FTS de fallback quando tag não basta
sleep_now() força o sono na hora
stats() contagem, score médio, fila de rascunho
